# 페이지 라우팅 구조

> 기준 코드: `frontend/src/App.tsx`, `frontend/src/main.tsx`  
> 라우팅 방식: `react-router-dom` (`BrowserRouter`, `useLocation`, `useNavigate`)  
> 최종 업데이트: 2026-02-24

---

## 1) 경로 맵

| Path | 화면 | 접근 조건 |
|---|---|---|
| `/login` | `<Login>` | 비로그인 기본 진입 |
| `/signup` | `<Signup>` | 회원가입 화면 |
| `/welcome` | `<WelcomePage>` | 로그인 직후 안내 화면 |
| `/dashboard` | `<DashboardTab>` | 로그인 필요 |

| `/data-input` | `<DataInput>` | 로그인 필요 |
| `/reports` | `<Reports>` | 로그인 필요 |
| `/analytics` | `<Analytics>` | 로그인 필요 |

---

## 2) 인증 가드/리다이렉트

`App.tsx`의 가드 규칙:

- 토큰 없음:
  - 공개 경로(`/login`, `/signup`) 외 접근 시 `/login`으로 이동
- 토큰 있음:
  - `/`, `/login`, `/signup` 접근 시 `/dashboard`로 이동
- 미정의 경로:
  - 토큰 유무에 따라 `/dashboard` 또는 `/login`으로 이동

---

## 3) Dashboard 탭 동기화

`activeTab`은 URL과 동기화됩니다.

- `/dashboard` -> `dashboard`
- `/dashboard/compare` -> `compare`
- `/dashboard/simulator` -> `simulator`
- `/dashboard/target` -> `target`

탭 변경 시 `navigateTo(view, tab)`가 path로 이동:

```ts
const buildPath = (view: ViewType, tab: TabType): string => {
  if (view === 'profile') return '/profile';
  if (view === 'data-input') return '/data-input';
  if (view === 'reports') return '/reports';
  if (view === 'analytics') return '/analytics';
  if (tab === 'compare') return '/dashboard/compare';
  if (tab === 'simulator') return '/dashboard/simulator';
  if (tab === 'target') return '/dashboard/target';
  return '/dashboard';
};
```

---

## 4) 로그인/회원가입 플로우

1. 로그인 성공 시 토큰 저장 후 `/welcome` 이동  
2. `WelcomePage` 자동 전환 또는 버튼 클릭으로 `/dashboard` 이동  
3. 로그아웃 시 토큰 제거 후 `/login` 이동

---

## 5) Header/프로필 진입

- 로고 클릭 -> `/dashboard`
- 헤더 탭 클릭 -> `/dashboard/*`
- 프로필 아이콘 -> `/profile`
- 프로필 내부 좌측 메뉴:
  - Dashboard -> `/dashboard`
  - Compare -> `/dashboard/compare`
  - Simulator -> `/dashboard/simulator`
  - Targets -> `/dashboard/target`

---

## 6) 히스토리/뒤로가기

- 브라우저 뒤로가기/앞으로가기는 `react-router` 히스토리로 자동 처리
- 기존 `window.history.pushState`/`popstate` 직접 제어는 제거됨
